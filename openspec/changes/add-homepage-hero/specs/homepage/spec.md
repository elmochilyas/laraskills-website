## MODIFIED Requirements

### Requirement: Homepage displays a hero section using foundation components
The system SHALL render a hero section on the homepage (`/`) that uses the foundation components (Button, Card, Section, PageContainer) and displays a clear value proposition with the product name, headline, description, call-to-action buttons, a terminal command card, and a trust line.

#### Scenario: Homepage uses PageContainer and Section
- **WHEN** the homepage is rendered
- **THEN** it SHALL use the `PageContainer` and `Section` components for layout

#### Scenario: Homepage displays product eyebrow
- **WHEN** the homepage is rendered
- **THEN** it SHALL display "Laraskills" as a small product eyebrow above the main headline

#### Scenario: Homepage displays main headline
- **WHEN** the homepage is rendered
- **THEN** it SHALL display the main headline: "Laravel engineering context for AI coding agents"

#### Scenario: Homepage displays description
- **WHEN** the homepage is rendered
- **THEN** it SHALL display a description explaining that Laraskills equips coding agents with Laravel-specific skills, rules, retrieval workflows, and MCP support

#### Scenario: Homepage includes a primary action button
- **WHEN** the homepage is rendered
- **THEN** it SHALL include a primary `Button` labeled "Install Laraskills" linking to installation instructions

#### Scenario: Homepage includes a secondary action button
- **WHEN** the homepage is rendered
- **THEN** it SHALL include a secondary `Button` labeled "View GitHub" linking to the GitHub repository

#### Scenario: Homepage displays a trust line
- **WHEN** the homepage is rendered
- **THEN** it SHALL display the text "Built for Laravel developers using AI coding agents" below the action buttons

#### Scenario: Two-column layout on desktop
- **WHEN** the homepage is rendered on a viewport at least 1024px wide
- **THEN** the text/CTA column SHALL appear on the left and the terminal card SHALL appear on the right

#### Scenario: Stacked layout on mobile
- **WHEN** the homepage is rendered on a viewport narrower than 1024px
- **THEN** the text/CTA column SHALL appear above the terminal card

### Requirement: Homepage has no placeholder HTML
The system SHALL NOT contain the original placeholder `<div>` with inline styles that was generated by `create-next-app`.

#### Scenario: Placeholder div is removed
- **WHEN** the homepage source is inspected
- **THEN** it SHALL NOT contain the original `create-next-app` placeholder markup

## ADDED Requirements

### Requirement: Terminal command card displays install command
The system SHALL display a terminal command card on the homepage showing the install command `npx laraskills@beta install` in a styled code block.

#### Scenario: Terminal card renders install command
- **WHEN** the homepage is rendered
- **THEN** a card with a styled terminal/command block SHALL display the text "npx laraskills@beta install"

#### Scenario: Terminal command is copyable
- **WHEN** the user clicks the copy button on the terminal card
- **THEN** the command text SHALL be copied to the user's clipboard and a "Copied" feedback indicator SHALL appear

### Requirement: Hero section uses dark developer-tool aesthetic
The system SHALL apply a premium dark developer-tool aesthetic to the hero section, matching the existing website design system.

#### Scenario: Section has subtle background glow
- **WHEN** the hero section is rendered
- **THEN** it SHALL optionally include a subtle radial-gradient glow effect behind the content

#### Scenario: No external UI libraries
- **WHEN** the homepage is inspected
- **THEN** all hero styling SHALL use existing CSS design tokens and Tailwind utilities without any external UI libraries
