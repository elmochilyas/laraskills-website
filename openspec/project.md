# Laraskills Website — Project Context

## Product

**Public brand name:** Laraskills

**Description:**
Laraskills is a Laravel-focused AI coding-agent operating layer and engineering knowledge system. It gives AI coding agents Laravel-specific skills, rules, retrieval workflows, and MCP support so they can work with deeper context and fewer guesses.

## Website Purpose

This repository is the standalone official website for Laraskills. It is the public vitrine/showcase of the project.

## Primary Goals

1. Present Laraskills as a serious open-source developer tool.
2. Clearly explain what Laraskills does and who it helps.
3. Make installation and onboarding easy.
4. Prepare the site for Vercel hosting.
5. Build strong SEO for search engines.
6. Build strong GEO / AI readability for LLMs, AI search engines, and coding agents.

## Tech Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- ESLint
- npm
- Vercel deployment

## Design Direction

- Premium developer-tool aesthetic.
- Dark technical interface.
- Laravel-inspired red accent.
- Clean typography.
- Terminal-style command examples.
- Knowledge graph / routing visual motifs.
- Minimal, fast, responsive, accessible.

## Branding Rules

- Public brand name is `Laraskills`.
- Do not use `laravel-ecc` as the public product name.
- If legacy package names or command names exist, treat them as implementation details and mark with TODO comments, not public marketing copy.

## Repository Boundary

This website is separate from the main Laraskills project repository. Do not duplicate the full knowledge base here.

## Quality Rules

- Work feature by feature through OpenSpec.
- Do not add fake metrics or testimonials.
- Do not add unnecessary UI libraries.
- Keep the site Vercel-ready.
